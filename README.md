# F1 Pit Stop Prediction ŌĆö ISYE 6740 Term Project (Group 090)

Predicting whether a Formula 1 driver will pit on the **next** lap, using only
information available at the end of the current lap.

## Problem

Binary classification. For every lap `L` of every race, predict whether that
driver enters the pit lane on lap `L+1`. Target column: `PitNextLap` (0/1).

The one-lap gap is deliberate ŌĆö predicting a stop that is already happening has
no practical value, while predicting it a lap early is actionable for a race
strategist deciding whether to cover a rival.

## Data

- **Source:** [FastF1](https://github.com/theOehrly/Fast-F1) ŌĆö lap-level F1 timing data
- **Coverage:** 2018ŌĆō2024, 149 races, 157,944 labeled laps after cleaning
- **Class balance:** ~3.2% positive (pit stops are rare events)

Raw data is not committed to this repo ŌĆö it is several hundred MB and is
regenerated by running the download script. FastF1 enforces a 500-request
hourly rate limit, so a full download takes several runs across a few hours.
The script is resumable and skips anything already saved.

## Pipeline

Run in this order:

| Script | Reads | Writes | Purpose |
|---|---|---|---|
| `download_data.py` | FastF1 API | `f1_laps_{year}.csv`, `f1_all_races.csv` | Download all races, 2018ŌĆō2025 |
| `build_target.py` | `f1_all_races.csv` | `f1_all_with_target.csv` | Build the `PitNextLap` label |
| `build_features.py` | `f1_all_with_target.csv` | `f1_all_features.csv` | Engineer six predictive features |
| `train_models.py` | `f1_all_features.csv` | console output | Train and compare five classifiers |
| `unsupervised_analysis.py` | `f1_all_features.csv` | `pca_clusters.png` | PCA, K-Means, spectral clustering, KDE |
| `make_figures.py` | `f1_all_features.csv` | four report figures | Generate charts for the write-up |

`test_fastf1.py` is the original single-race proof of concept, kept for reference.

## Features

| Feature | Description |
|---|---|
| `TireDegDelta` | Seconds off the driver's own best clean lap on the current tire set |
| `AnyCautionThisLap` / `CautionJustStarted` | Safety car or VSC active; whether it *just* appeared |
| `WetConditions` | Derived from tire compound (INTERMEDIATE/WET) as a weather proxy |
| `RaceProgress` / `LapsRemaining` | Position in the race, as a fraction and as an absolute count |
| `StopsSoFar` | Pit stops already made this race |
| `TyreLife`, `Position`, `Stint` | Available directly from FastF1 |

Two data-quality issues are handled explicitly:

- **In-laps, out-laps, and caution laps are excluded from pace baselines.** Under
  a safety car the whole field runs far slower, which an uncorrected tire
  degradation metric misreads as catastrophic wear. Leaving these in caused the
  model to score *below random* on caution laps specifically.
- **Race distance is computed per race, never per driver.** Using a driver's own
  maximum lap would encode their retirement as "100% through the race."

## Validation

Split by **year**, never randomly ŌĆö laps within a race are heavily correlated,
so a random split would leak.

- **Train:** 2018ŌĆō2022 (107,866 laps)
- **Validate:** 2023 (23,950 laps) ŌĆö used only to select decision thresholds
- **Test:** 2024 (26,128 laps) ŌĆö scored once

Class imbalance handled via `class_weight='balanced'` (or explicit sample
weights for AdaBoost, which lacks that parameter). ROC AUC is the headline
metric since it requires no threshold choice.

## Results

### Supervised

| Model | ROC AUC | F1 |
|---|---|---|
| **Random Forest** | **0.7998** | 0.1722 |
| SVM (RBF) | 0.7849 | 0.1525 |
| AdaBoost | 0.7754 | 0.1478 |
| Logistic Regression | 0.7333 | 0.1501 |
| Gaussian Naive Bayes | 0.7067 | 0.1129 |
| Naive baseline (tire age > k) | ŌĆö | 0.0848 |

Non-linear models clearly outperform linear ones. `RaceProgress` illustrates
why: its single-feature AUC is 0.4468 (worse than a coin flip) yet it is the
Random Forest's most important feature. Pit rate rises through the middle of a
race then collapses at the end ŌĆö a shape no linear model can represent, but
one a tree splits trivially.

### Ablation ŌĆö the central experiment

Removing the caution feature set from the Random Forest:

| | With caution features | Without |
|---|---|---|
| All test laps | **0.7998** | 0.7706 |
| Caution laps only | **0.6266** | 0.6132 |

The caution features improve performance both overall and, importantly, on the
caution laps themselves.

### Unsupervised

K-Means (k=4, chosen by silhouette scan) was given the same features but never
saw `PitNextLap`. It independently isolated a 2,994-lap cluster defined almost
entirely by a just-started caution, with a **14.0% pit rate against a 3.2%
baseline** ŌĆö matching the supervised finding. It also separated a wet-weather
cluster at 4.8%.

Two methods, one supervised and one blind to the label, converge on the same
conclusion.
